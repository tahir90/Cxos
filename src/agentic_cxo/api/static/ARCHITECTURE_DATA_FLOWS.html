1. Issues & Gaps — Premium Sale Blockers

Comprehensive list of known blockers beyond PPT quality. These prevent selling at a premium price to founders.

1.1 Critical (blocks core value)

PPT/PDF output quality below benchmark

Generated presentations are "basic and useless" vs Claude's output. Root causes: generic synthesis, wrong title handling, lack of structured layouts (data_metrics, comparison_table, benefits_risks, warning_callout), no chart support.

Two separate architectures, not unified

CoFounder/Chat path: Uses PlannerTool → PlanExecutor → tools (researcher, presentation_generator). Used by web UI /chat, /chat/stream.
Cockpit path: Uses Orchestrator (Cockpit) → ScenarioEngine or direct agent dispatch. Used by CLI and /scenarios API.
Scenarios (14 pre-defined workflows like Cash-Flow Guardian) are NOT reachable from the main chat UI. Founders using the chat never see scenario execution.

Scenarios require Vault data; Vault is often empty

Scenario steps use vault_query to pull context (invoices, CRM, Slack). Without prior document ingestion, the Vault has no data. Founders must proactively ingest files — there is no guided onboarding flow.

1.2 High (degrades perceived value)

Title slide shows wrong/generic text

Plan intent ("Create a professional research presentation on...") is passed as title instead of the actual topic ("Excessive Use of Agentic AI on Human Brain Development").

Research → Synthesis gap

Researcher finds good sources (EY, arxiv, MIT). Synthesis prompt does not enforce: specific study names + years, statistics, definition blocks, comparison rows, benefits/risks lists. Output is generic bullets.

Slide spec rarely picks rich layouts

LLM chooses content_bullets over data_metrics, comparison_table, benefits_risks, warning_callout because the outline lacks structured content for those layouts.

No chart support in presentation generator

Benchmark has bar charts (e.g. Global AI User Growth). Our presentation.py does not render charts.

Live connectors require manual setup

Salesforce, Slack, HubSpot, etc. exist but need OAuth/credentials. No clear "connect your stack" wizard. Founders hit credential errors when scenarios expect live data.

1.3 Medium (polish & trust)

CXO-to-CXO only in Plan path

NEED_INPUT_FROM (e.g. CFO asks CMO) works only when the plan includes consult_agent steps. Direct presentation flow (researcher → presenter) never invokes CXOs.

Methodology Designer/Auditor/Review not tuned

Agents exist but prompts may not push hard enough for study citations, metrics, and layout structure. Validation loops can pass mediocre output.

PPT QA depends on LibreOffice + Poppler

Vision-based QA requires system deps. Fails silently or skips on Railway/Docker if not installed. No graceful fallback messaging.

Plan creation is LLM-only

If LLM fails or API key missing, no plan is created. User gets 503 or generic fallback. No offline mode.

2. Scenario 1: PPT/Research Presentation (Full Plan Flow)

User prompt: "Create a PPT on 'excessive use of agentic AI on human brain development' with gmg.com branding. Do your research."

End: Response to user with document download link (or error).

User → CoFounder Agent → IntentRouter + Planner
PlannerTool.create_plan() → LLM

Plan: 7 steps (example)

Group A: research (×3), Group B: synthesize, Group C: consult CD, Group D: generate, Group E: validate
PlanExecutor → Researcher (×3 topics)

Methodology Designer → Research → Review Agent → Methodology Auditor (validation loop)

Output: findings, sources

Synthesize step → Methodology Designer → LLM synthesis

Review Agent + Methodology Auditor on outline. Retry if fail.

Output: markdown outline

CD (Creative Director) → visual brief, templates
Generate step → PresentationGeneratorTool

Methodology Designer (doc) → Slide spec (LLM) → generate_pptx → PPT QA (vision) → Review/Audit on output → Retry if fail

Validate step → CD.validate_output
CoFounder → User (combined_summary + document_ready URL)

CXO Involvement

CDConsulted for visual direction, layout advice. Validates final output.
CFO, CMO, etc.Only if plan includes consult_agent step (LLM decides). Typically NOT in PPT-only plan.

Communication Flow

Events streamed to UI: plan_created, step_group_start, step_start, step_progress, step_complete, source_found, agent_consulted, document_ready, plan_complete.

3. Scenario 2: Simple CXO Question

User prompt: "What's our burn rate and runway?"

End: Natural language response from CFO (or routed agent).

User → CoFounder → IntentRouter

Planner.should_plan() → needs_planning: false (simple question)

IntentRouter.route() → CFO (keyword: burn rate, runway → finance)
ToolExecutor.decide_and_execute() → CostAnalyzerTool (or similar)

Vault query for financial data. If no data → generic or error.

_respond_naturally() → LLM (with CXO persona + tool result)
CoFounder → User

CXO Involvement

CFORouted by keywords. Tool (CostAnalyzer) does work. LLM formats response with CFO persona.
Other CXOsNone. No cross-agent.

4. Scenario 3: Pre-defined Scenario (Cash-Flow Guardian)

User prompt: (CLI) cxo scenario run cfo-cash-flow-guardian OR (API) POST /scenarios/cfo-cash-flow-guardian/run

End: ScenarioResult with step outputs, possibly pending approvals.

User / API → Cockpit.run_scenario() → ScenarioEngine.execute()

Scenario: Cash-Flow Guardian. Steps: cfg-1 (expense spikes) → cfg-2 (audit) → cfg-3 (runway) → cfg-4 (briefing)

Step cfg-1 → Vault.query("non-payroll expenses...") → AgentCFO

Vault returns chunks (or empty). CFO agent reasons, produces AgentAction. RiskAssessor, ApprovalGate if HIGH.

Steps cfg-2, cfg-3 → AgentCFO (depends_on: cfg-1 output)
Step cfg-4 → AgentCFO (compile briefing)
ScenarioEngine → ScenarioResult

CXO Involvement

CFOAll 4 steps. Each step: vault_query → agent.reason() → action. No PlanExecutor.
Other CXOsNone. Scenario is single-agent (CFO).

⚠️ Critical Gap

This flow is NOT triggered from the chat UI. Founders using the web app never run scenarios. Scenarios require CLI or explicit /scenarios API call.

5. Scenario 4: Cross-CXO Communication

User prompt: "Create a market expansion plan. Consider legal and tax implications."

End: Response combining CMO + CLO + CFO insights.

User → Planner

Plan includes: consult_agent CMO, consult_agent CLO, consult_agent CFO (or multi-step)

CMO → LLM → Response

If CMO output contains: NEED_INPUT_FROM: CLO | What are the liability caps for our target market?

_handle_cross_agent_request() → CLO → LLM (CLO persona)

CLO response appended to CMO content. Validation loop (Review + Auditor). Retry if fail.

Synthesize → User

Communication Table

CMO → CLOCMO output includes "NEED_INPUT_FROM: CLO | <question>". PlanExecutor parses, calls CLO, injects response.
CLO → CMOCLO response merged into CMO content. Single round-trip (no CLO asking back).

⚠️ Limitation

Cross-CXO only happens when (a) plan includes consult_agent, and (b) LLM output contains NEED_INPUT_FROM. Not guaranteed. Direct presentation flow never uses CXOs.

6. Scenario 5: Direct Presentation (No Plan)

User prompt: "Create a ppt on AI trends"

Condition: Planner.should_plan() returns needs_planning: false, BUT _extract_presentation_request() finds topic.

End: PPT delivered. Bypasses full plan.

User → CoFounder → topic extracted: "AI trends"
Researcher.execute(topic="AI trends") → findings, sources

No Methodology Designer. No synthesis step. Outline = research.summary or findings joined.

PresentationGeneratorTool.execute() → slide_spec (if CD) → generate_pptx

No Methodology Designer for doc gen. No Review/Audit on output. No PPT QA in this path (varies by code path).

CoFounder → User

CXO Involvement

CDSlide spec uses CD if available. No explicit CD consult step.
CFO, CMO, etc.None. Zero CXO involvement.

⚠️ Quality Impact

This path produces lower-quality PPTs: no synthesis validation, no methodology brief, no multi-source consolidation structure. Same "basic and useless" output.

7. Summary & Recommendations

Architecture Decisions Needed

  1. Unify Chat and Cockpit: Should scenarios be runnable from chat? ("Run the Cash-Flow Guardian scenario")
  2. Kill or fix direct presentation path: Either route all PPT requests through full plan, or add Methodology Designer + synthesis to direct path.
  3. Title flow: Pass topic explicitly to presentation generator, not plan.intent.
  4. Synthesis prompt overhaul: Require study names, years, stats, definitions, comparison rows, benefits/risks.
  5. Slide spec tuning: When outline has numbers → data_metrics. When before/after → comparison_table. When pros/cons → benefits_risks.
  6. Chart support: Add bar-chart rendering in presentation.py.
